# 少样本思维链（Few-shot-CoT）：给模型看几个带推理的例子

## 一句话结论

少样本思维链（Few-shot-CoT）是在Prompt中提供几个带推理过程的示例，让模型"模仿"这种逐步思考的方式，效果比Zero-shot-CoT更稳定可控，是生产环境的首选方案。

---

## 1. Why：背景与痛点

### 1.1 业务背景
- **Zero-shot-CoT不够稳**：效果波动大，有时候有用有时候没用
- **格式不统一**：推理过程的格式每次都不一样，难以解析
- **生产环境要求高**：需要稳定、可预期的输出

### 1.2 技术背景
- **示例的力量**：给模型看几个好例子，比说一百句"要这么做"都管用
- **模仿学习**：大模型擅长模仿Prompt中的模式
- **格式锚定**：示例中的推理格式会引导模型用同样的格式输出

---

## 2. What：概念与边界

### 2.1 核心定义
**少样本思维链（Few-shot-CoT）**：在Prompt中提供 k（通常 2~5）个「问题-推理-答案」的完整示例，让模型模仿这种逐步推理的风格。

### 2.2 示例对比

**Zero-shot-CoT**：
```
问：一个盒子里有红、蓝、绿三种球，红球5个，蓝球3个，绿球2个，随机拿一个，不是红球的概率是多少？
让我们一步步思考。
答：...（格式不确定）
```

**Few-shot-CoT**：
```
示例1：
问：小明有10颗糖，给了小红3颗，又买了5颗，现在有多少颗？
推理：小明一开始有10颗，给了小红3颗后剩7颗，又买了5颗，7+5=12。
答案：12

示例2：
问：超市苹果5元一斤，买3斤需要多少钱？
推理：苹果5元一斤，买3斤就是5×3=15元。
答案：15

新问题：
问：一个盒子里有红、蓝、绿三种球，红球5个，蓝球3个，绿球2个，随机拿一个，不是红球的概率是多少？
推理：
答案：
```

## 3. How：原理与实践

### 3.1 写好Few-shot-CoT示例的要点

#### 要点1：示例格式一致
```
✅ 好：
推理：XXX。
答案：YYY。

❌ 不好：
推理：XXX
答案YYY
```

#### 要点2：示例质量 > 数量
```
✅ 2个高质量示例 > 5个低质量示例
```

#### 要点3：覆盖典型情况
```
示例应该覆盖：
- 常见情况
- 边界情况
- 易错情况
```

---

### 3.2 技术选型

| 维度 | Zero-shot-CoT | Few-shot-CoT |
|------|---------------|--------------|
| **示例数** | 0 | 2~5 |
| **效果** | 不稳定 | 稳定 |
| **格式控制** | 弱 | 强 |
| **成本** | 低 | 中等 |
| **适用** | 快速验证 | 生产环境 |

## 5. 面试官追问清单

### Q1：Few-shot-CoT需要多少个示例？
**回答要点**：
- 通常 2~5 个就够了
- 质量 > 数量
- 可以做ablation study确定最佳数量

### Q2：Few-shot-CoT和微调怎么选？
**回答要点**：
- Few-shot-CoT：快速、无需训练、灵活
- 微调：效果更好、Prompt短、推理快
- 先Few-shot-CoT，效果不够再微调

---

## 6. 总结提纲

1. **定义**：Few-shot-CoT = 几个带推理的示例 + 新问题
2. **效果**：比Zero-shot-CoT更稳定，格式更统一
3. **要点**：示例格式一致、质量>数量、覆盖典型情况
4. **适用**：生产环境、格式要求严格、高准确率要求
5. **选型**：先Zero-shot，不行再Few-shot，还不行再微调
