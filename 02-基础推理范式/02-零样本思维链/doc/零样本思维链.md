# 零样本思维链（Zero-shot-CoT）：一句话激活推理能力

## 一句话结论

零样本思维链（Zero-shot-CoT）是CoT的极简变体，仅需在问题后加一句"让我们一步步思考"（Let's think step by step），无需任何示例就能激活模型的推理能力，简单粗暴但效果显著。

---

## 1. Why：背景与痛点

### 1.1 业务背景
- **Few-shot-CoT需要写示例**：每次新任务都要写几个带推理过程的示例，费时间
- **示例质量影响大**：示例写得不好，效果反而下降
- **快速验证需求**：想快速看看CoT对某个任务有没有用，不想写示例

### 1.2 技术背景
- **Few-shot-CoT的成功**：证明了显式推理的威力
- **大模型的内在能力**：模型其实已经会逐步推理，只是需要"引导"
- **极简主义的胜利**：有时候一句话就够了

### 1.3 约束与目标
| 维度 | 目标 | 约束 |
|------|------|------|
| **简单性** | 最简单的CoT方式 | 只能用一句话 |
| **效果** | 比Standard好 | 可能不如Few-shot-CoT |
| **成本** | 最低 | - |

---

## 2. What：概念与边界

### 2.1 核心定义
**零样本思维链（Zero-shot-CoT）**：不提供任何示例，仅在问题后追加一句触发词（如"让我们一步步思考"），引导模型自动展开推理链。

### 2.2 示例对比

**Standard Prompting**：
```
问：一个农场有鸡和兔子共30只，脚共80只，鸡和兔子各几只？
答：鸡20只，兔子10只
```

**Zero-shot-CoT**：
```
问：一个农场有鸡和兔子共30只，脚共80只，鸡和兔子各几只？
让我们一步步思考。
答：假设全是鸡，应该有60只脚，多了20只脚，每只兔子比鸡多2只脚，所以兔子10只，鸡20只。
```

### 2.3 常用触发词

| 语言 | 触发词 |
|------|--------|
| 中文 | 让我们一步步思考 |
| 中文 | 请详细描述你的推理过程 |
| 中文 | 先思考，再回答 |
| 英文 | Let's think step by step |
| 英文 | Let's work this out step by step |

**结论句**："让我们一步步思考"（或英文"Let's think step by step"）是最经典、效果最稳定的触发词。

---

## 3. How：原理与实践

### 3.1 原理：为什么一句话就能激活推理？

**结论句**：大模型在预训练阶段已经见过大量"问题-推理-答案"的数据，Zero-shot-CoT只是把这个模式"唤醒"了。

#### 关键机制
1. **模式匹配**：触发词激活了模型预训练中见过的"逐步推理"模式
2. **生成空间约束**：把模型从"直接给答案"拉到"先推理再答案"的轨道上
3. **无需示例**：模型内部已有足够的推理知识，不需要外部示例

---

### 3.2 使用流程

```
1. 明确问题
   ↓
2. 追加触发词
   ↓
3. 调用模型
   ↓
4. 获取推理过程+答案
```

---

### 3.3 技术选型：Standard vs Zero-shot-CoT vs Few-shot-CoT

| 维度 | Standard | Zero-shot-CoT | Few-shot-CoT |
|------|----------|---------------|--------------|
| **示例数** | 0 | 0 | 1~5 |
| **触发词** | 无 | 有 | 可有可无 |
| **优点** | 最快最省 | 简单、比Standard好 | 效果最好 |
| **缺点** | 复杂任务差 | 效果不稳定、格式不统一 | 需要写示例 |
| **成本** | 最低 | 低 | 中等 |
| **适用** | 简单问题 | 快速验证、中等复杂度 | 生产环境 |

---

### 3.4 实现要点

#### 要点1：触发词的位置
```
✅ 正确：问题在前面，触发词在后面
问：XXX
让我们一步步思考。

❌ 不推荐：触发词在前面
让我们一步步思考。
问：XXX
```

#### 要点2：可以多次触发
```
进阶用法：
问：XXX
让我们一步步思考。
（模型生成推理）
现在，根据以上推理，给出最终答案。
```

## 4. 优缺点分析

### 4.1 优点

| 优点 | 说明 |
|------|------|
| **最简单** | 无需写示例，加一句话就行 |
| **适用广** | 几乎所有需要推理的任务都能试试 |
| **成本低** | 比Few-shot-CoT省Token |
| **快速验证** | 想看看CoT有没有用，先用这个试 |

---

### 4.2 缺点

| 缺点 | 说明 |
|------|------|
| **效果不稳定** | 有时候有用，有时候没用，看模型心情 |
| **格式不统一** | 推理过程的格式可能每次都不一样 |
| **不如Few-shot** | 通常效果比Few-shot-CoT差一些 |

---

## 5. 适用场景

### 5.1 推荐使用场景

| 场景 | 说明 |
|------|------|
| **快速验证** | 想快速看看CoT对某个任务有没有用 |
| **原型阶段** | 产品早期，还不确定要不要做CoT |
| **成本敏感** | Token预算有限，不想用Few-shot |
| **简单推理** | 推理步骤不多的任务 |

---

### 5.2 不推荐使用场景

| 场景 | 说明 |
|------|------|
| **生产环境** | 效果不稳定，不适合核心业务 |
| **格式要求严格** | 推理格式不统一，难以解析 |
| **高准确率要求** | 通常不如Few-shot-CoT |

## 7. 面试官追问清单

### Q1：Zero-shot-CoT为什么只用一句话就有用？
**回答要点**：
- 模型预训练时见过大量"问题-推理-答案"的数据
- 触发词激活了这个模式
- 不是"教"模型推理，是"唤醒"它已有的能力
- 小模型可能没用，大模型才管用

### Q2：Zero-shot-CoT和Few-shot-CoT怎么选？
**回答要点**：
- 快速验证 → Zero-shot-CoT
- 生产环境 → Few-shot-CoT
- 先试Zero-shot，不行再上Few-shot
- Zero-shot成本低，Few-shot效果稳

### Q3：触发词有什么讲究？
**回答要点**：
- "让我们一步步思考"最经典
- 英文用"Let's think step by step"
- 放在问题后面，不是前面
- 可以试试不同的触发词，看哪个效果好
- 有的模型对触发词敏感，有的不敏感

---

## 8. 总结提纲

1. **定义**：Zero-shot-CoT = 问题 + 一句触发词，无需示例
2. **触发词**："让我们一步步思考" / "Let's think step by step"
3. **原理**：激活模型预训练中见过的推理模式
4. **优点**：最简单、成本低、快速验证
5. **缺点**：效果不稳定、格式不统一
6. **适用**：快速验证、原型阶段、成本敏感
7. **不适用**：生产环境、严格格式、高准确率
8. **选型**：先Zero-shot，不行再Few-shot
