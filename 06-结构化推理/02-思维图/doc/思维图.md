# 思维图 (Graph of Thoughts, GoT)

## 一句话结论 (TL;DR)
思维图 (GoT) 是将 LLM 推理过程建模为**有向图 (DAG)** 的框架。它突破了 CoT 的线性限制和 ToT 的树状分支限制，允许思维节点之间的**聚合 (Aggregation)**、**合并 (Merging)** 和**循环优化**，使模型能够像人类大脑一样，在多个思路之间交叉验证并汇总信息。

---

## 1. Why：为什么需要思维图？

### 1.1 从“线”到“树”再到“图”
- **思维链 (CoT)**：线性路径，无法回头，一步错步步错。
- **思维树 (ToT)**：分支探索，允许回溯，但各分支之间是孤立的，无法共享中间成果。
- **思维图 (GoT)**：**网状协作**。它认为解决复杂问题往往需要将 A 路径的发现与 B 路径的结论结合起来，生成更优的 C 方案。

### 1.2 架构本质：思维的可转换性
GoT 的核心在于定义了**思维变换 (Thought Transformations)**，将 LLM 推理过程看作是对信息的加工流：
- **聚合**：将多个中间结论汇总。
- **细化**：对一个不成熟的结论进行迭代修正。
- **生成**：基于现有节点产生新节点。

---

## 2. What：核心组件与操作

![思维图](../../../images/GoT.png)

### 2.1 核心组件
1. **思维节点 (Thought Nodes)**：表示推理过程中的一个中间状态或结论。
2. **有向边 (Directed Edges)**：表示思维之间的依赖关系。
3. **控制器 (Controller)**：调度 LLM 进行各种“思维变换”。

### 2.2 三大关键变换操作
- **聚合 (Aggregation)**：例如，在写论文摘要时，分别生成引言、方法、结果的子摘要，然后“聚合”成最终摘要。
- **合并 (Merging)**：将两个互补的思路合并。
- **循环/迭代 (Iteration)**：对一个节点进行自反馈，直到满足评估标准。

## 3. How：实现架构

### 3.1 逻辑流程
```text
[节点1: 思路A] ----\
                   \
                    --> [节点3: 聚合后的新方案] --> [最终答案]
                   /
[节点2: 思路B] ----/
```

### 3.2 智能座舱应用举例
**场景：多维度的复杂行程规划**
- **任务**：策划一个为期 3 天的上海商务+旅游行程，要求包含特定会议、避开高峰、且包含三家不同菜系的顶级餐厅。
- **GoT 推理过程**：
    - **节点 1**：并行生成 3 天的商务会议排期。
    - **节点 2**：并行搜索符合要求的餐厅及位置。
    - **节点 3 (变换)**：将会议排期与餐厅位置进行**地理聚合**，计算通勤效率。
    - **节点 4 (迭代)**：发现第 2 天午餐太远，触发**细化操作**，重新选择餐厅。
    - **节点 5**：最终生成综合行程图。

## 4. 面试官追问清单

1. **Q: GoT 相比 ToT 的核心优势在哪里？**
   - **A**: 核心优势是**信息共享与路径协同**。ToT 的每个分支都是独立的“孤岛”，而 GoT 允许将不同路径的优点“合流”。对于需要综合多方信息的任务（如长文本摘要、多目标优化），GoT 效率更高。

2. **Q: GoT 会不会导致搜索空间爆炸？**
   - **A**: 会。因为图的连接比树更复杂。因此 GoT 必须配合强有力的**评分器 (Scorer)** 和**剪枝策略**。通常只在关键节点进行聚合。

3. **Q: GoT 在 Prompt 工程中如何落地？**
   - **A**: 落地通常依赖**状态管理框架**。需要程序化地记录每个生成的片段（节点），并根据预设逻辑（如：如果生成了 3 个子模块，则触发聚合 Prompt）来调度下一次 LLM 调用。

---

## 5. 总结提纲
- [ ] **网状结构**：支持多路径合并与协作。
- [ ] **变换操作**：聚合、细化、合并是 GoT 的灵魂。
- [ ] **信息增益**：通过组合中间结论减少重复推理。
- [ ] **复杂任务**：专为需要全局综合判断的场景设计。
