# 思维树 (Tree-of-Thought, ToT)

## 一句话结论 (TL;DR)
思维树 (ToT) 将大语言模型的概率生成能力与经典的启发式搜索算法（如 BFS、DFS）相结合。它不再将推理视为单向的线性链条，而是将其建模为一个可回溯、可剪枝的树状状态空间，使模型能够在复杂问题中进行全局搜索与多步规划。

---

## 1. Why：为什么需要思维树？

### 1.1 线性推理的局限性
在传统的思维链 (CoT) 中，模型遵循“单向快思考”模式。一旦推理路径在中间步骤出现偏差，由于缺乏回溯机制，模型往往会“将错就错”，导致最终答案完全偏离。

### 1.2 架构本质的跃迁
思维树的架构本质在于：**将 LLM 的概率生成能力与经典搜索算法框架解耦**。
- **不再将 LLM 视为“黑盒”**：而是将其视为一个**状态机生成器 (State Machine Generator)**。
- **搜索算法作为调度器**：由算法决定何时生成新状态、何时评估状态、何时回溯或剪枝。

---

## 2. What：核心组件与流程
在思维树模式下，Agent不再是一位写作“一气呵成”的作家，而是一位“步步为营”的棋手。其核心机制的本质是将LLM的概率生成过程重构为思维空间中的启发式搜索过程。这一过程依赖于3个核心原语——分解(Decomposition)、生成(Generation)与评估(Evaluation)的紧密配合。
![思维树](../../../images/ToT.png)

### 2.1 四大核心模块
1. **思维分解 (Thought Decomposition)**：将复杂问题拆解为多个离散的中间步骤（思维单元）。一个“思维”需足够小使模型能生成多样样本，足够大使模型能评估进展。为确保思维的多样性，通常需要设置较高的Temperature值，或使用专门的Prompt模板。
2. **思维生成 (Thought Generator)**：根据当前状态生成多个候选的下一步思维。
3. **状态评估器 (State Evaluator)**：作为启发式函数，评估当前状态对最终解决问题的“前景”（如评分：Sure/Maybe/Impossible）。
4. **搜索算法 (Search Algorithm)**：全局调度器，使用广度优先搜索 (BFS)、深度优先搜索 (DFS) 或束搜索 (Beam Search) 在思维空间中游走。

### 2.2 流程示意图
```text
      [初始状态]
      /    |    \
   [思维A] [思维B] [思维C]  <-- 思维生成
     |      |       |
   (评估) (评估)  (剪枝X)  <-- 状态评估
     |      |
   [下一步] [下一步]         <-- 搜索算法调度
```

## 3. How：实现要点与案例
思维树模式从最初的“投票机制”起步，逐步发展为支持“深度搜索”的结构，最终演进为融合价值函数的“蒙特卡洛树搜索”​。
### 3.1 实现策略
- **采样策略**：在“思维生成”阶段，可以通过设置不同的 `temperature` 或 `top_p` 来获取多样化的路径。
- **剪枝机制**：评估器发现某个分支为 `Impossible` 时，立即停止该分支的搜索，节省算力。

### 3.2 智能座舱应用举例
**场景：多约束路径规划与任务调度**
- **问题**：用户要求“今天下午 3 点前要去接孩子，路上要买束花，还要顺便给车充个电，推荐最省时间的路线”。
- **ToT 推理过程**：
    - **层级 1**：生成多个首选目的地顺序（先买花 vs 先充电）。
    - **层级 2**：针对每个顺序，计算不同路况下的预估时间。
    - **层级 3**：评估器检查约束条件（是否能在 3 点前完成）。
    - **决策**：剪掉超时的分支，在剩余分支中选择最优路径。

## 4. 面试官追问清单

1. **Q: ToT 与 Self-Consistency (自一致性) 有什么区别？**
   - **A**: Self-Consistency 是生成多条完整的链然后投票；ToT 是在推理的**中间步骤**进行分支、评估和搜索。ToT 具有更强的局部控制和纠错能力。

2. **Q: ToT 的算力成本如何优化？**
   - **A**: 关键在于评估器的准确性。一个高效的评估器可以尽早剪枝。此外，可以使用束搜索 (Beam Search) 限制每一层保留的状态数量，平衡搜索深度与宽度。

3. **Q: ToT 适合所有场景吗？**
   - **A**: 不适合。对于简单逻辑或直觉类任务，CoT 甚至 Zero-shot 就足够了。ToT 专为具有**巨大搜索空间、复杂约束或需要前瞻性规划**的任务（如 24 点游戏、创意写作规划、复杂调度）设计。

---

## 5. 总结提纲
- [ ] **解耦**：生成与搜索分离。
- [ ] **状态机**：LLM 生成状态，算法驱动流转。
- [ ] **评估**：启发式评估决定搜索方向。
- [ ] **回溯**：允许在死胡同处回退，重新选择。
