# Agent 协作模式：辩论（对抗）——真理越辩越明

## 一句话结论 (TL;DR)
辩论模式通过构建一个包含“正方-反方-法官”的三角架构，利用角色间的结构性张力(Structural Tension)迫使信息在对抗中提炼与校正，实现生成能力与判断能力的物理解耦，有效降低大模型的幻觉风险。

---

## 1. Why：背景与痛点

### 1.1 业务背景
在处理高风险决策（如自动驾驶路径规划、医疗诊断建议、金融投资策略）时，单体 Agent 的输出具有不确定性。

### 1.2 核心痛点
- **单体幻觉**：单体 Agent 容易产生自我确认偏见，编造看似合理但事实错误的论据。
- **无法自证**：模型很难发现自己推理路径中的逻辑漏洞。
- **信任边界**：对于关键决策，用户需要看到多维度的权衡过程而非单一的结果。

### 1.3 架构转型
从“**生成式**”向“**判别式**”范式转变。不再盲目相信生成内容，而是通过对抗博弈来筛选最经得起推敲的方案。

---

## 2. What：核心组件与架构

### 2.1 三角架构 (Triangle Architecture)
架构师需要构建一个封闭的“辩论场”(Arena)，核心组件如下：

| 组件 | 角色 | 设定关键词 | 职责 |
| :--- | :--- | :--- | :--- |
| **正方 (Proponent)** | 方案发起者 | 乐观、建设性、防御性 | 生成初始方案，提供支持性论据，对质疑进行反驳。 |
| **反方 (Opponent)** | 质量把控者 | 批判性、怀疑性、攻击性 | 质疑论点，识别潜在幻觉，提出反例或边界情况。 |
| **法官 (Judge)** | 决策仲裁者 | 客观、严谨、高推理能力 | 主持流程，汇总观点，作出最终裁决。必须由强模型（如 GPT-4/5）担任。 |

### 2.2 核心价值
**能力解耦**：将“生成能力”（正/反方）与“判断能力”（法官）在架构层面实现物理分离。正如法庭上律师的陈词不影响法官的公正。

---

## 3. How：流程与实现要点

### 3.1 辩论流程
1. **开篇立论**：正方基于 User Prompt 生成初始方案 A。
2. **第一轮攻辩**：反方针对方案 A 寻找漏洞，生成质疑报告 B。
3. **第二轮反驳**：正方针对报告 B 进行辩解，修正方案得到 A'。
4. **结辩陈词**：双方整理最终观点。
5. **法官判决**：法官基于全过程记录，分析逻辑合理性，给出最终裁决方案。

### 3.2 实现要点
- **Prompt 工程**：为正反方注入截然不同的性格特征。
- **轮次控制**：通常 2-3 轮对抗即可达到信息提炼的收益平衡点。
- **异构模型**：推荐正反方使用不同厂商的模型，利用模型多样性（Diversity）增加博弈深度。

---

## 4. 优化与改进

- **并行采样**：多个正方生成不同方案，由多个反方进行交叉射击。
- **动态角色切换**：根据辩论进度，让法官要求双方互换立场，进一步挖掘逻辑漏洞。
- **外部工具辅助**：反方可调用 Web Search 工具实时查证正方的论据。

---

## 5. 智能座舱举例：复杂行程决策

### 5.1 场景映射
- **用户需求**：“我今天想去远郊露营，但电量剩余 30%，推荐路线。”
- **正方 (规划者)**：推荐最近的露营点 A，认为 30% 电量勉强够到，途中可下坡动能回收。
- **反方 (安全员)**：质疑露营点 A 海拔高，上坡耗电远超预期；指出途中唯一的充电桩可能在维修；提出备选露营点 B。
- **法官 (中枢)**：分析海拔模型数据，查证充电桩实时状态。裁决：接受反方建议，前往露营点 B 以保证安全冗余。

## 6. 面试官追问清单

1. **Q: 为什么不用单模型 Self-Correction (自我修正)？**
   - **A**: 单模型往往存在“确认偏见”，它倾向于证明自己是对的。对抗模式引入了独立的“反向目标”，结构性张力更强，纠错率显著提高。
2. **Q: 这种架构会带来延迟(Latency)增加，如何权衡？**
   - **A**: 辩论模式适用于“**高价值/慢决策**”场景。对于简单的控制指令（如开关车窗），应使用零样本或路由模式；对于涉及安全、大额支出或长远规划，增加的延迟是必要的成本。
3. **Q: 法官模型必须比正反方模型强吗？**
   - **A**: 是的。判别能力通常高于生成能力，但法官需要理解复杂的逻辑链条。如果法官能力较弱，容易被双方的“激情陈词”误导。

---

## 7. 总结提纲
- [ ] **封闭辩论场**：通过角色设定引入结构性张力。
- [ ] **生成与判断分离**：正反方负责生成，法官负责决策。
- [ ] **降低幻觉**：对抗博弈迫使逻辑漏洞暴露。
- [ ] **场景适配**：适用于高风险、高复杂度的关键决策。
