## 基础提示工程

### 零样本提示 (Zero-shot Prompting)
零样本提示是指在不向模型提供任何参考示例的情况下，直接通过指令让模型执行特定任务。这是对模型预训练和指令微调成果的直接调用，类似于人类的"系统1"思维（快思考），依赖直觉和概率预测，而非严谨的逻辑推演。

### 少样本提示 (Few-shot Prompting)
少样本提示通过在Prompt中提供k个"输入-输出"对（Demonstrations），引导模型理解任务模式。这是一种"激活"而非"学习"，示例充当了锚点，缩小了模型在生成时的搜索空间。对于格式要求极其严格的任务，哪怕只有一个示例（One-shot）也能带来质的飞跃。

## 基础推理范式

### 思维链 (Chain-of-Thought, CoT)
使模型在提供最终答案之前输出推理步骤，从而改进复杂推理任务。CoT本质上是把隐式推理显式化：把复杂任务拆成若干可检查的子结论，降低一步到位时的跳跃。

**核心价值**：
- 提升多步推理任务（数学、逻辑、规划）的稳定性
- 让错误更容易被定位与纠正
- 驱动"先分析后作答"的工作流

**注意**：展示出来的推理不等同于真实内部机制，且可能包含不必要或不可靠的细节。

#### 零样本思维链 (Zero-shot-CoT)
通过简单的提示如"让我们一步步思考"来触发逐步思考。

#### 少样本思维链 (Few-shot-CoT)
使用少量示例引导模型进行逐步推理。

#### 自一致性思维链 (Self-Consistency, CoT-SC)
生成多个推理路径并使用投票机制确定最终答案，从而提高准确性。这是一种"让模型多想几遍再投票"的策略，解决单次生成的偶然性与脆弱性问题。

#### 复杂思维链 (Complex CoT)
为需要较长推理序列的复杂任务而设计。

## 工具使用与行动交互

### ReAct (Reason + Act)
一种交替进行推理和行动的范式：
1. 推理当前状态
2. 决定行动（工具调用）
3. 观察结果
4. 重复直到达成目标

ReAct构建了"推理"与"行动"的飞轮，让模型能在与现实世界的交互中动态思考与学习。当模型不确定时，它可以去"查资料"（调用Wikipedia API），用事实证据来支撑推理，减少了幻觉。

### ToolFormer
训练模型在文本生成中插入API/工具调用，实现无缝工具使用。

### 函数调用 / 工具调用 (Function Calling / Tool Calling)
主要提供商（GPT、Claude、Qwen等）使用的结构化函数/工具调用方法。

### WebGPT / 网络搜索增强
结合网络搜索能力与推理，基于最新信息做出明智决策。

## 规划与执行

### 规划并执行 (Plan-and-Execute)
两阶段方法：
1. 任务的全局规划
2. 子任务的顺序执行

适用于需要结构化执行的长周期任务。

### 规划-解决-推理 (Plan-Solve-Reason, PSR)
1. 规划阶段
2. 解决子问题
3. 从解决方案中综合推理

### 从易到难提示 (Least-to-Most Prompting)
将复杂问题分解为更简单的子问题，按难度递增顺序解决。

### 自主规划 (Self-Planning)
使模型能够自主生成执行计划，无需外部指导。

### Plan-Then-Action (PTA)
两阶段框架，第一阶段利用高级LLMs将CoT提炼成紧凑的高层指导用于监督微调，第二阶段引入指导感知的强化学习方法，联合优化最终输出和细粒度CoT推理。

## 反思、迭代与自我修正

### 自我精炼 / 自我改进 (Self-Refine / Self-Improve)
迭代精炼过程：
1. 生成初始输出
2. 自我批判
3. 精炼
4. 重复直到满意

### 自我修正 (Self-Correction)
模型在推理过程中自主检测和纠正错误。

### 反思 (Reflexion)
行动后反思机制：
1. 执行行动
2. 反思结果
3. 更新未来行动的决策策略

### 迭代提示 (Iterative Prompting)
多轮精炼方法，每轮迭代都提高输出质量。

## 结构化推理

### 思维树 (Tree-of-Thought, ToT)
在树状结构中探索多个推理路径：
1. 思维分解：一个"思维"需足够小使模型能生成多样样本，足够大使模型能评估进展
2. 思维生成：根据思维空间维度选择采样策略
3. 状态评估器：评估状态对最终解决问题的"前景"
4. 搜索算法：使用BFS、DFS或其他搜索算法

ToT将"推理"从一条线扩展为一棵树，在关键步骤上生成多个候选想法（分叉），并对这些分支进行评估、剪枝与继续展开，直到找到更优解。适合规划、搜索、策略博弈、复杂约束问题等。

### 思维图 (Graph-of-Thought, GoT)
更灵活的基于图的推理结构，允许思维间的复杂连接。每个思维节点既能生成多个子节点，亦可接收多个父节点的输入，实现信息的汇聚与重组。这种支持分支（多个输出）和聚合（多个输入）的特性，模拟了动态规划等高级推理模式。

### 思维算法 (Algorithm-of-Thought, AoT)
将算法方法（DFS/BFS等）应用于决策过程。

### 顺序思维 (Sequential Thinking)
严格的线性逐步推理方法。

## 结构化输出与决策框架

### 程序辅助语言模型 (PAL, Program-Aided Language Models)
将自然语言推理与代码生成结合，实现精确计算和逻辑运算。

### ToT + BFS/DFS
将思维树与搜索算法集成，以优化决策路径。

### 有向无环图执行 (DEPS, Directed Acyclic Graph Execution)
在有向无环图结构中执行任务，实现高效的依赖管理。

### PDDL风格规划
使用AI规划研究中的符号化规划方法进行结构化任务分解。

## Agent级决策系统

### AutoGPT / BabyAGI
自主agent，能够：
- 将目标分解为子任务
- 执行长期计划
- 维护过去行动的记忆
- 适应动态环境

### 生成式Agent (Generative Agents)
类人agent，具有：
- 持久记忆系统
- 反思能力
- 规划机制
- 动态交互模式

### 任务驱动的自主Agent
针对特定任务优化的闭环决策系统，具有持续反馈集成。

### 增强记忆的决策制定
利用短期和长期记忆的决策框架：
- 工作记忆用于即时上下文
- 情节记忆用于过去经验
- 语义记忆用于一般知识
- 程序记忆用于技能和例程

## Agent 协作模式 (Agent Collaboration Patterns)

### 辩论（对抗） (Debate / Adversarial)
类似于正反双方在法庭上的辩论，通过模型间的相互批判与质询，使真理在交锋中变得更加清晰明了，有效减少单一模型的偏见与幻觉。

### 委托（层级） (Delegation / Hierarchy)
如同经理指挥各个工人，采用“中心节点-执行节点”的层级结构，将复杂目标层层分解，以实现高度组织化的执行。

### 路由（分发） (Routing / Dispatch)
就像医院的分诊台，能够将请求根据任务特征（如领域、复杂度）准确地分配到最适合其需求的专门 Agent 或模型。

### 群体（涌现） (Swarm / Emergence)
仿佛蚁群迁徙，即便没有明确的全局指挥者，通过大量 Agent 间的简单局部交互，也能展现出惊人的整体组织性和效率。

## 最新推理范式

### 思维矩阵 (Matrix of Thought, MoT)
2025年提出的新型高效思维结构。MoT通过"列-单元通信"机制在水平和垂直维度探索问题，使LLM能够积极参与多策略和深度思考，同时减少列单元内思维节点的冗余，从而增强LLM的推理能力。

**核心特性**：
- 二维探索：同时在水平和垂直维度进行推理
- 列-单元通信：实现思维节点间的高效信息交换
- 事实校正机制：利用RAG检索的知识图谱三元组和原文构建知识单元，纠正错误答案

**优势**：推理时间仅为基线方法的14.4%，同时在24点游戏、问答评估等任务上优于现有方法。

### 超树规划 (HyperTree Planning, HTP)
2025年提出的新型推理范式，用于构建超树结构的规划大纲以实现有效规划。超树结构使LLM能够通过灵活运用分治策略进行层次化思考，有效分解复杂推理步骤，适应多样化约束，并以有序方式管理多个不同的子任务。

**核心特性**：
- 层次化思考：通过超树结构实现分治策略
- 迭代精炼：自主规划框架通过迭代精炼和扩展超树结构的规划大纲完成规划过程
- 自反思机制：引导模型在整个过程中修订和改进推理

**性能表现**：在TravelPlanner基准测试中，使用Gemini-1.5-Pro实现了最先进的准确性，性能比o1-preview提高了3.6倍。

### 智能体推理 (Agentic Reasoning)
2025年牛津大学提出的新框架，通过整合外部工具智能体来增强大型语言模型的推理能力。与传统仅依赖内部推理的LLM推理方法不同，智能体推理框架结合了网络搜索、代码执行和结构化推理上下文记忆。

**核心组件**：
- **Mind Map智能体**：通过构建结构化知识图谱追踪逻辑关系，提升演绎推理能力
- **网络搜索智能体**：具备精准信息检索能力，根据推理进度动态调整搜索关键词
- **代码智能体**：支持Python代码即时执行，实现实时计算验证

**优势**：
- 动态知识增强：根据推理阶段动态调整检索策略
- 逻辑可视化追踪：知识图谱让思考过程具备可解释性
- 实时计算验证：用计算结果反向验证理论推导
- 记忆强化学习：连续处理相关问题时自动合并知识节点

**性能表现**：在博士水平科学推理任务（GPQA）上准确率提升35%，特别是在需要跨学科整合的生物学问题上，得分从62%跃升至79%。

### Think Node-by-Node
受思维导图、流程图等可视化工具启发的基于图的推理框架。该方法利用标准的图构建和渲染库，不需要微调，只需要模型的原生编码能力。

## 实现考虑因素

在实现这些范式时，需要考虑：

1. **计算效率** - 准确性与资源使用的平衡
   - CoT-SC、ToT等范式计算成本较高，需根据任务复杂度选择
   - MoT在保持准确性的同时显著降低推理时间

2. **错误处理** - 处理失败的健壮机制
   - 推理过程的可观察性对于错误定位至关重要
   - 自一致性和自我修正机制可提高鲁棒性

3. **可扩展性** - 范式在任务复杂性增加时的表现
   - 层次化方法（HTP、MoT）更适合复杂多步骤任务
   - 图结构（GoT）支持更灵活的推理拓扑

4. **可解释性** - 理解决策如何做出
   - 注意：CoT显示的推理过程不一定是模型的真实内部机制
   - 研究显示Claude 3.7 Sonnet的CoT忠实度仅约25%
   - Mind Map等可视化方法可提升可解释性

5. **集成** - 结合多种范式以增强能力
   - 可根据任务阶段动态切换推理策略
   - Agentic Reasoning展示了多智能体协同的潜力
